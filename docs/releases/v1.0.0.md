# v1.0.0 Release Notes

Release date: July 2026

## Overview

SSG-Hackathon v1.0 is the first stable release of the invite-only collaborative workstation for hackathon teams.

## What's Included

- **Workspace Management** — Create and manage hackathon workspaces with templates
- **Planning** — Objectives, milestones, deliverables, requirements, risks, decisions
- **Ideas** — Brainstorming, voting, and concept refinement
- **Research** — Collaborative research with source tracking and verification
- **Tasks** — Full task management with status, priority, checklists, and assignment
- **Notes** — Personal and shared notes with categories
- **Files** — File upload and folder organization
- **Relationships** — Universal relationship system connecting all modules
- **Analytics** — Workspace health, trends, and comprehensive reporting
- **Automation** — Rule-based automation with 25 trigger types and 11 action types
- **Archive** — Workspace archival with retrospectives and lessons learned
- **Collaboration** — Comments, mentions, reviews, and review requests
- **Notifications** — 22 notification types with real-time polling
- **GitHub Integration** — OAuth and PAT-based connection with issues, PRs, commits
- **Integration Centre** — Extensible integration framework
- **Admin Centre** — Owner-only platform management
- **Discovery** — Hackathon event discovery and pipeline tracking
- **Submission** — Submission preparation with deliverables and checklists
- **Settings** — Profile, account, team, security, and storage management
- **Mission Control** — Central dashboard with workspace overview and health metrics

## Deployment

See `docs/deployment/first-run.md` for deployment and initial setup instructions.

## Key Technical Details

- 60 database tables across 32 migrations
- 62 application routes
- Next.js 16.2.10 with App Router
- Supabase Auth with GitHub OAuth
- Tailwind CSS v4 dark theme
- TypeScript strict mode

## Known Limitations

- RLS policies are permissive (private invite-only platform)
- No server-side middleware beyond proxy.ts
- Some FK constraints intentionally omitted for flexibility
- PAT tokens encrypted, OAuth tokens in session memory
- No E2E tests in test suite (unit tests only)
